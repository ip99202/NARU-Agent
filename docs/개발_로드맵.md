## 📐 시스템 아키텍처

```
[Chainlit UI]
     │ 사용자 입력
     ▼
[LangGraph Orchestrator]
  ├─ Planner Node   ← 의도 파악 & 계획 수립
  ├─ Router         ← 분기 결정 (Tool 호출 / 승인 요청 / 완료)
  ├─ Executor Node  ← MCP Tool 호출
  └─ Approval Node  ← Human-in-the-loop 승인 대기
     │               (Chainlit 내장 Action Button 활용)
     ▼
[MCP Server (NARU Wrapper)]
  └─ NARU API 호출
        └─ 세션 쿠키 기반 인증 (로그인 선행)
```

## 🛠 기술 스택 (확정)

| 역할 | 기술 | 이유 |
|---|---|---|
| **UI** | **Chainlit** | LLM 채팅 UI 특화, LangGraph 공식 통합, async 네이티브, Human-in-the-loop 내장 지원 |
| **LLM** | Claude 3.5 Sonnet | 복잡한 Tool Use & 코딩 능력 우수 |
| **Orchestrator** | LangGraph | 순차 워크플로우, 루프, 분기 처리 필수 |
| **프로토콜** | MCP (Model Context Protocol) | NARU 레거시 시스템과 LLM 표준 연결 |
| **프롬프트 전략** | ReAct + CoT | 동적 계획 수립(Dynamic Planning) 필수 |
| **메모리** | LangGraph Checkpointer (MemorySaver → SqliteSaver) | LangGraph 자체 상태관리. 메시지 히스토리가 State에 포함되어 별도 메모리 불필요 |
| **RAG** | ❌ 사용 안 함 | - |

> **Chainlit을 선택한 이유 (vs Streamlit)**
> - Streamlit은 매 인터랙션마다 전체 스크립트를 재실행 → LangGraph `interrupt()` 상태 유지 불가
> - Chainlit은 async 기반으로 LangGraph와 자연스럽게 통합, `cl.AskActionMessage`로 승인 버튼 UI 즉시 구현 가능

### Phase 1 — 로그인 & 세션 관리 (Day 1~2) ⭐ 최우선

**목표:** NARU API 로그인 성공 확인 + 세션 유지 메커니즘 구현

모든 NARU API 호출은 로그인 후 발급된 세션 쿠키가 필요합니다.
이것이 없으면 아무 API도 호출 불가능하므로 **가장 먼저 구현하고 테스트**합니다.

```python
# mcp_server/tools/auth.py
import httpx
from config import NARU_BASE_URL, NARU_USER_ID, NARU_USER_PW

# 세션을 전역으로 관리 (MCP Server 수명주기와 동일)
_session: httpx.AsyncClient | None = None

async def get_session() -> httpx.AsyncClient:
    """로그인된 세션을 반환. 없으면 자동 로그인."""
    global _session
    if _session is None:
        _session = httpx.AsyncClient(base_url=NARU_BASE_URL)
        response = await _session.post(
            "/api/verify-code/pre-login",
            json={"userId": NARU_USER_ID, "userPw": NARU_USER_PW}
        )
        response.raise_for_status()
        # 세션 쿠키는 httpx.AsyncClient가 자동으로 유지
        print(f"✅ NARU 로그인 성공: {response.status_code}")
    return _session
```

**로그인 테스트 스크립트 (`test_login.py`):**

### Phase 3 — LangGraph 오케스트레이터 (Day 5~7)

**목표:** 계획 → 실행 → 승인 루프 구현

#### 3-1. State Schema

```python
# graph/state.py
from typing import Annotated, Any
from langgraph.graph.message import add_messages
from pydantic import BaseModel

class AgentState(BaseModel):
    messages: Annotated[list, add_messages]   # 대화 히스토리 (State에 내장)
    next_step: str = ""                        # 다음 행동 방향
    pending_tool_calls: list = []              # 승인 대기 Tool 호출
    is_approved: bool | None = None            # 사용자 승인 여부
    user_feedback: str = ""                    # 거절 시 피드백
    final_answer: str = ""                     # 최종 답변
```

> **메모리 전략:** LangGraph `Checkpointer` 기반으로 메시지 히스토리가 `AgentState.messages`에 자동 누적됨.
> `ConversationSummaryBufferMemory` 같은 LangChain 레거시 메모리 객체는 **사용하지 않음**.
> 세션 간 지속성이 필요하면 `MemorySaver` → `SqliteSaver`로 교체.

#### 3-2. 그래프 조립

```python
# graph/graph.py
from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver

def build_graph():
    builder = StateGraph(AgentState)
    builder.add_node("planner", planner_node)
    builder.add_node("executor", executor_node)
    builder.add_node("approval", approval_node)
    builder.set_entry_point("planner")

    builder.add_conditional_edges("planner", router, {
        "call_tool":     "executor",
        "need_approval": "approval",
        "final_answer":  END,
    })
    builder.add_edge("executor", "planner")
    builder.add_conditional_edges("approval",
        lambda s: "executor" if s.is_approved else "planner",
        {"executor": "executor", "planner": "planner"}
    )

    # 개발: MemorySaver / 운영: SqliteSaver("naru_agent.db")
    return builder.compile(
        checkpointer=MemorySaver(),
        interrupt_before=["approval"]
    )
```

### Phase 5 — Chainlit UI (Day 8~9)

**목표:** 채팅 인터페이스 + Human Approval 버튼 UI

```python
# app.py
import chainlit as cl
from langgraph.types import Command
from graph.graph import build_graph

graph = build_graph()

@cl.on_chat_start
async def on_start():
    cl.user_session.set("thread_id", cl.context.session.id)

@cl.on_message
async def on_message(message: cl.Message):
    thread_id = cl.user_session.get("thread_id")
    config = {"configurable": {"thread_id": thread_id}}

    result = await graph.ainvoke(
        {"messages": [("user", message.content)]},
        config=config
    )

    # Human Approval 처리
    if result.get("__interrupt__"):
        interrupt_data = result["__interrupt__"][0].value
        actions = [
            cl.Action(name="approve", value="yes", label="✅ 승인"),
            cl.Action(name="reject",  value="no",  label="❌ 거절"),
        ]
        await cl.AskActionMessage(
            content=interrupt_data.get("message", "신청을 진행할까요?"),
            actions=actions
        ).send()
    else:
        await cl.Message(content=result.get("final_answer", "")).send()

@cl.action_callback("approve")
async def on_approve(action: cl.Action):
    thread_id = cl.user_session.get("thread_id")
    config = {"configurable": {"thread_id": thread_id}}
    result = await graph.ainvoke(Command(resume={"approved": True}), config=config)
    await cl.Message(content=result.get("final_answer", "신청이 완료되었습니다.")).send()

@cl.action_callback("reject")
async def on_reject(action: cl.Action):
    thread_id = cl.user_session.get("thread_id")
    config = {"configurable": {"thread_id": thread_id}}
    result = await graph.ainvoke(Command(resume={"approved": False}), config=config)
    await cl.Message(content="신청을 취소했습니다. 다시 시도해 주세요.").send()
```

## ⚠️ 구현 시 주의사항

1. **세션 만료 처리:** NARU 세션이 만료되면 `get_session()`에서 자동 재로그인 로직 필요
   ```python
   async def get_session():
       global _session
       # 401 응답 시 세션 초기화 후 재로그인
       if _session:
           test = await _session.get("/api/health")
           if test.status_code == 401:
               _session = None
       ...
   ```

2. **Checkpointer 선택:**
   - 개발: `MemorySaver()` (인메모리, 재시작 시 초기화)
   - 운영: `SqliteSaver.from_conn_string("naru_agent.db")` (파일 영속)

3. **멀티 사용자:** `thread_id`를 Chainlit 세션 ID(`cl.context.session.id`)로 자동 부여 → 사용자별 대화 격리

4. **Write API는 반드시 Human Approval:** `draft_*`, `request_*` 계열 Tool 호출 감지 시 `need_approval` 분기 필수
