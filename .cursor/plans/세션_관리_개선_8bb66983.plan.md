## Fix 2B: probe 제거 + 401-only 재로그인 (auth.py)

`get_session()`에서 매번 probe 하는 로직 제거. 대신 실제 API에서 401 받으면 재로그인 후 반환하는 `refresh_session()` 함수 추가.

**변경 전 get_session() 흐름:**

```
_session 없음 → login()
_session 있음 → probe GET /api/bizcomm/chrgr/my → 401이면 재로그인
```

**변경 후 get_session() 흐름:**

```
_session 없음 → login()
_session 있음 → 그냥 반환
```

**새 함수 refresh_session() 추가:**

```python
async def refresh_session() -> httpx.AsyncClient:
    """401 수신 시 호출 — 세션 닫고 재로그인 후 반환."""
    global _session
    if _session:
        await _session.aclose()
    _session = await login(...)
    return _session
```

각 tool 파일은 현재 `get_session()`만 호출하고 있어 tool 파일은 수정 불필요. 401 재시도 로직은 auth.py 내부에서 처리할 필요 없고, 실제 API 호출 시 `raise_for_status()`가 있어서 401이 터지면 에이전트가 재시도하게 됨. 따라서 `refresh_session()`은 노출만 해두고 필요 시 tool에서 명시 호출 가능.

> 단순화 방향: get_session()에서 probe만 제거. 401 핸들링은 기존 `raise_for_status()` 동작에 맡김.
